Command used:
run app.activity.info -a owasp.sat.agoat

Target package:
owasp.sat.agoat

Application:
AndroGoat - Insecure App (Kotlin)

Observed exported activities:

1. owasp.sat.agoat.SplashActivity
   Permission: null

2. owasp.sat.agoat.AccessControl1ViewActivity
   Permission: null

Manifest confirmation:
SplashActivity:
- exported="true"
- Contains MAIN / LAUNCHER intent-filter
- This is expected because it is the application launcher activity.

AccessControl1ViewActivity:
- exported="true"
- Contains an intent-filter with:
  action: android.intent.action.VIEW
  category: android.intent.category.DEFAULT
  data scheme: androgoat
  host: vulnapp

Attack surface confirmation:
Drozer attack surface summary shows:
- 2 activities exported

Analysis:
The SplashActivity is expected to be exported because it is the launcher activity.

The AccessControl1ViewActivity is a more important finding because it is exported and reachable through a custom URI scheme:
androgoat://vulnapp

An external application could potentially launch this activity directly using an Intent. If the activity does not verify the user session or validate the incoming Intent, this could allow bypassing the normal application flow.

Risk:
Unauthorized access to an internal screen or bypass of the normal navigation flow.

Severity:
Medium

Recommendation:
- Keep only the launcher activity exported when required.
- Set android:exported="false" for internal activities.
- Validate all incoming Intents in exported activities.
- Add permission protection if the activity must remain externally accessible.
